§1 · Type Scale — Live Specimen

Why not pure φ? — A pure φ scale from 16px gives h2 at 110px, h1 at 178px. That's mathematically correct but typographically absurd for web layout. Print masters (Tschichold, Müller-Brockmann) established that display type above 60px needs a decelerating scale — proportional to the square root of φ for large sizes. Our scale uses φ steps for body→h4, then moderates for h2→display. The result: mathematically proportional at every step, but optically harmonious across all sizes.

§2 · Spacing Scale — Visual Ruler

The 4px Atom. Every spacing value is a multiple of 4px. This is the fundamental grid unit — the smallest meaningful gap the eye can detect at screen resolution. The φ progression from this base creates: icon gaps → inline → component → section → hero. Each step is 1.618× the previous. The body font-size (16px = space-4) appears at step 4 — this alignment of type and space is intentional and creates invisible harmonic resonance.

§3 · Golden Grid — Layout Proportions

The 1100px container. This number was not chosen arbitrarily. 1100 / φ = 680px — the ideal prose reading width (confirmed by readability research: 60–70 characters per line at 18px = ~650–700px). So the container IS the golden expansion of the optimal reading width. When you center 680px text in 1100px, the remaining 420px creates two margins of 210px each. 420 / 680 = 0.618 = 1/φ. Every empty space is mathematically proportional to the filled space.

§5 · Shadow System — Atmospheric Depth

Two-layer shadow principle. Real light creates two types of shadow: ambient (scattered light from the whole sky — large, soft, low opacity) and key (directional sun/lamp — small, sharp, higher opacity). Single-layer CSS shadows look fake. Our system uses: ambient = large offset + large blur + low opacity; key = small offset + tight blur + medium opacity. The shadow tint uses our espresso color (#1A1410) not pure black — warm shadow tint stays on-brand and avoids the cold look of rgba(0,0,0).

§6 · The Constitutional Rules

Typography Laws

1. Never use more than 2 type sizes adjacent without a hierarchy break.

2. Body text: Effra only. Headings: Quincy only. Never mix roles.

3. Line length: 52–72 characters for body. Wider = harder to read.

4. Display type always gets tracking-tight or tighter. Never track display loose.

5. Uppercase labels always get tracking-wide minimum. Unspaced caps are illegible.

Spacing Laws

1. All spacing values must be from the scale. No arbitrary px values.

2. Space between heading and body: space-4 (= 1rem = the type base).

3. Card padding: space-6 standard. Never below space-5, never above space-7.

4. Section vertical rhythm: space-8. This is non-negotiable. Sections need breath.

5. Gap between type and CTA button: space-5 minimum. Give the call-to-action air.

Layout Laws

1. 2-column splits: always 61.8/38.2 or 50/50. No 40/60 or 30/70.

2. Narrow prose content: max 680px. Never wider. Reading comfort is a law.

3. Sections alternate light/dark. Never two adjacent dark or two adjacent light sections.

4. Grid gaps = space-5 (cards) or space-6 (major elements). Consistent, never ad-hoc.

5. Container max-width: 1100px. Never increase. Whitespace on wide screens is a feature.

The φ Cheat Sheet
