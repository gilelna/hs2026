§1 · Breakpoint Map

Content-first breakpoints, not device-first. We don't break at "iPhone 14 width." We break when the content breaks: when the golden 2-column layout can no longer hold both columns comfortably, when a 5-column magazine strip becomes illegible, when body text drops below 52 characters per line. The breakpoints are set at those exact content collapse points. Gutenberg's native breakpoints (600px, 781px) are too early for our 1100px container — so we extend with custom CSS in blocks.css.

md · 768 – 1023px · iPad

→ Golden 2-col layout activates (61.8/38.2).

→ Container padding: 2.5rem (40px).

→ 3-col cards grid activates.

→ Full nav visible. Floating pill nav.

→ Section padding: ~96px.

→ Magazine: 3 columns.

lg · 1024 – 1279px · Laptop

→ Container approaches max-width.

→ Container padding: 3.5rem (56px).

→ 5-col magazine activates.

→ Hero image full height.

→ Section padding: ~110px.

→ All grids at full column count.

xl · 1280 – 1535px · Desktop

→ Full 1100px container, centered.

→ Container padding: 4rem (64px).

→ Type at comfortable reading size.

→ Generous negative space on sides.

→ Section padding: clamp max ~128px.

→ Optimal experience target.

2xl · 1536px+ · Wide Screen

→ Container locked at 1100px.

→ Auto side margins grow — intentional whitespace.

→ Full-width section BGs still span 100vw.

→ Type at maximum clamp values.

→ Never increase container. Whitespace is content.

→ Section backgrounds fill edge to edge.

§2 · Fluid Type · Live Specimen — Resize Window

One formula, all screens. Each type size uses a single CSS clamp() value — no media queries for type ever. The formula interpolates linearly between a minimum (at 320px viewport) and maximum (at 1536px). This means type is always perfectly proportional at every in-between size. Breakpoints only control layout (columns, stacking) never type size.

§3 · Gutenberg Block Architecture

Every section = Cover or Group + Columns + Groups. The outer wrapper is always a wp:group or wp:cover set to align: full (100vw background). Inside sits a nested group set to align: wide (1100px container). Inside that: wp:columns for multi-column layouts. Inside each column: wp:group as card wrapper. Custom responsive CSS lives in css/blocks.css.
